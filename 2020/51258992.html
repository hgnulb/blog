      class Vehicle {
    int maxSpeed = 120;
}

class Car extends Vehicle {
    int maxSpeed = 180;

    void display() {
        System.out.println("Maximum Speed: " + maxSpeed);
        System.out.println("Maximum Speed: " + super.maxSpeed);
    }
}

class Test {
    public static void main(String[] args) {
        Car small = new Car();
        small.display();
    }
}

      Maximum Speed: 180 Maximum Speed: 120

      class Person {
    void message() {
        System.out.println("This is person class");
    }
}

class Student extends Person {
    void message() {
        System.out.println("This is student class");
    }

    void display() {
        message();
        super.message();
    }
}

class Test {
    public static void main(String args[]) {
        Student s = new Student();
        s.display();
    }
}

      this 和 super 注意事项

      • 由于 this 和 super 都是在实例化阶段调用，所以不能在静态方法和静态代码块中使用 this 和 super 关键字。

      • 一个实例变量可以通过 this 赋值另一个实例变量；一个实例方法可以通过 this 调用另一个实例方法；甚至一个构造方法都可以通过 this 调用另一个构造方法。

      • 如果 this 和 super 指代构造方法，则必须位于方法体的第一行。

        换句话说，在一个构造方法中，this 和 super 只能出现一次且只能出现一次，否则在实例化对象时，会因子类调用到多个父类的构造方法而造成混乱。

      • 任何类在创建之初，都有一个默认的空构造方法，它是 super() 的一条默认通路，构造方法的参数列表决定了调用通路的选择。

        如果子类指定调用父类的某个构造方法，super 就会不断往上溯源，如果没有指定，则默认调用 super()。

        如果父类没有提供默认的构造方法，子类在继承时就会出现编译错误。

        如果父类坚持不提供默认的无参构造方法，那么必须在本类的无参构造方法中使用 super 显示调用父类的有参构造方法。
